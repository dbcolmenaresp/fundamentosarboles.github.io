MOTIVACIÓN
¿Por qué necesitamos Árboles?
Lo lineal vs. Lo jerárquico
Las estructuras lineales (listas, pilas, colas) modelan relaciones 1 a 1.
El mundo real está lleno de relaciones 1 a N (jerarquías).

📁 Sistemas de Archivos

Un directorio contiene muchos archivos y subdirectorios. Cada subdirectorio contiene más archivos. Rama natural.

🏢 Organizaciones

Un CEO supervisa a varios gerentes, cada gerente a varios empleados. Estructura piramidal en Organigramas.

🧬 Herencia en POO

Una clase padre puede tener múltiples clases hijas, que a su vez pueden tener más hijas. Jerarquía de clases.

🌐 Documentos HTML/XML

El DOM (Document Object Model) es un árbol donde cada elemento puede contener elementos anidados.

⚡ Conclusión: Los árboles son la estructura natural para representar datos con relaciones de pertenencia y contención.

FUNDAMENTOS
Definición Formal de Árbol

Árbol: Un conjunto finito de nodos que satisface las siguientes condiciones:

  • ✔ Existe un nodo especial llamado raíz (no tiene padre).
  • ✔ Los demás nodos se particionan en subárboles disjuntos.
  • ✔ Cada nodo tiene un grado = número de hijos.
  • ✔ No se presentan ciclos cerrados en su estructura.

📐 Terminología Esencial

  • Raíz: Nodo superior, origen de todo.
  • Padre / Hijo: Relación directa de adyacencia.
  • Hermano (Sibling): Mismo padre.
  • Hoja (Leaf): Grado 0 → sin hijos.
  • Nodo Interno: Tiene al menos un hijo.
  • Nivel / Profundidad: Distancia desde la raíz (raíz = nivel 0).
  • Altura: Máximo nivel de cualquier nodo.
● Raíz
  ├── ● Hijo
  │  ├── ● Hoja
  │  └── ● Hoja
  └── ● Hijo
      ├── ● Hoja
      └── ● Hoja
PROPIEDAD IMPORTANTE Un árbol con N nodos tiene exactamente N − 1 aristas (conexiones).
TAXONOMÍA
Clasificación de los Árboles

Los árboles se clasifican según restricciones estructurales y propósito.

🌲 Generales

Sin límite de hijos. Modelan jerarquías puras.

🌳 Binarios

Máximo 2 hijos (izquierdo y derecho). Base de muchos algoritmos.

⚖️ Autobalanceados

Garantizan altura O(log n) mediante rotaciones.

💾 Árboles B

Multirrama optimizados para discos y bases de datos.

🔤 Tries

Árboles de prefijos para búsqueda eficiente de strings.

Cada tipo resuelve un problema específico de eficiencia, memoria o acceso.

TIPOS • NIVEL 1
🌲 Árbol General

Concepto

Un árbol donde un nodo puede tener cualquier número de hijos. No hay restricción de grado.

Propiedad esencial: Representa jerarquías puras sin restricciones artificiales.

¿Dónde se usa?

  • 📁 Sistemas de archivos (carpetas con N archivos).
  • 🏛️ Organigramas empresariales.
  • 📊 Árboles sintácticos en compiladores (AST).
COMPLEJIDAD Operaciones básicas: O(n) en el peor caso (si está degenerado).

Ejemplo Visual

  ├── ● Gerente Ventas
  │  ├── ● Vendedor A
  │  ├── ● Vendedor B
  │  └── ● Vendedor C
  ├── ● Gerente IT
  │  ├── ● Dev Frontend
  │  ├── ● Dev Backend
  │  └── ● DBA
  └── ● Gerente Finanzas
      ├── ● Contador
      └── ● Analista
TIPOS • NIVEL 2
🌳 Árbol Binario

Concepto

Árbol donde cada nodo tiene como máximo 2 hijos: izquierdo y derecho.

Propiedad fundamental: Su estructura restringida permite recorridos ordenados y algoritmos recursivos muy elegantes.

Subtipos importantes

  • Árbol Binario Lleno (Full): Cada nodo tiene 0 o 2 hijos.
  • Árbol Binario Completo (Complete): Todos los niveles están llenos, excepto quizás el último (llenado de izquierda a derecha).
  • Árbol Binario Perfecto: Todos los nodos internos tienen 2 hijos y todas las hojas están al mismo nivel.
USO Base para BST, Heaps, Árboles de expresión y más.

Ejemplo de Árbol Binario

● 10
  /      \
● 5      ● 15
 /  \     /  \
● 2  ● 7  ● 12  ● 20

Cada nodo tiene ≤ 2 hijos. Aquí es también un BST.

TIPOS • BINARIOS
🔍 Árbol Binario de Búsqueda (BST)

Concepto

Un árbol binario con una invariante de orden:

Regla de oro: Para todo nodo N:
• Todos los nodos en el subárbol izquierdo tienen valor < N.valor.
• Todos los nodos en el subárbol derecho tienen valor > N.valor.

Intuición de eficiencia

Esta propiedad permite descartar la mitad de los datos en cada paso, similar a la búsqueda binaria en un arreglo ordenado.

  • O(log n) en promedio.
  • O(n) en el peor caso (si los datos están ordenados y el árbol se convierte en una lista).

Ejemplo de Búsqueda

● 50
  /      \
● 30      ● 70
 /  \     /  \
● 20  ● 40  ● 60  ● 80

🔹 Buscar 40: 50→30→40 (3 pasos).

🔹 Buscar 10: 50→30→20→NULL (no existe).

🔹 Recorrido In-orden: 20,30,40,50,60,70,80 → ¡ordenado!

TIPOS • AUTOBALANCEADOS
⚖️ Árbol AVL

Concepto

El primer árbol de búsqueda autobalanceado (1962, Adelson-Velsky y Landis).

Invariante fundamental: Para cada nodo, la diferencia de altura entre su subárbol izquierdo y derecho es -1, 0 o +1.
(Factor de equilibrio = altura(izq) − altura(der)).

¿Cómo mantiene el balance?

Cuando una inserción o eliminación rompe el equilibrio, se aplican rotaciones (simple o doble) para restaurarlo.

VENTAJA Búsquedas O(log n) garantizado.
DESVENTAJA Más costoso en inserciones/eliminaciones (más rotaciones).

Ejemplo: Inserción que causa rotación

// Insertar: 10, 20, 30
10 → 10 → 10 (factor -2)
                 \          \
                 20       20
                            \
                            30

// Rotación izquierda sobre 10
      20
    /    \
10     30

🔄 Rotación restaura el factor de equilibrio.

TIPOS • AUTOBALANCEADOS
🔴⚫ Árbol Rojo-Negro

Concepto

Árbol binario de búsqueda con un bit extra de color (rojo o negro) en cada nodo.

Las 5 reglas invariantes:
  1. Cada nodo es rojo o negro.
  2. La raíz es negra.
  3. Todas las hojas (NULL) son negras.
  4. Un nodo rojo no puede tener hijos rojos (no hay dos rojos seguidos).
  5. Todos los caminos desde un nodo a sus hojas tienen el mismo número de nodos negros (altura negra).

Intuición

Estas reglas garantizan que el árbol esté aproximadamente balanceado (altura ≤ 2·log₂(n+1)), con menos rotaciones que AVL.

¿Dónde se usa?

  • 📦 TreeMap y TreeSet en Java.
  • 📦 std::map, std::set en C++.
  • 📦 Implementación de diccionarios en varios lenguajes.
  • 📦 Planificación de procesos en kernels de Linux (CFS).
VENTAJA Menos rotaciones que AVL → mejor para inserciones/eliminaciones frecuentes.
COMPLEJIDAD Búsqueda, Inserción, Eliminación: O(log n) garantizado.
TIPOS • BINARIOS
⛰️ Heap (Montículo Binario)

Concepto

Un árbol binario completo (todos los niveles llenos, excepto quizás el último, llenado de izquierda a derecha) con una propiedad de prioridad.

Propiedad de orden (Heap Property):
• Max-Heap: El valor de cada nodo es ≥ que el de sus hijos (la raíz es el máximo).
• Min-Heap: El valor de cada nodo es ≤ que el de sus hijos (la raíz es el mínimo).

Intuición de eficiencia

No es un árbol de búsqueda (no hay orden entre izquierda y derecha). Su poder está en acceder al extremo (max/min) en O(1).

  • Inserción: Colocar al final y "flotar" (sift up) → O(log n).
  • Eliminar extremo: Reemplazar con el último y "hundir" (sift down) → O(log n).

Aplicaciones Estrella

  • Colas de prioridad (PriorityQueue en Java/Python).
  • Algoritmo de Dijkstra (camino más corto en grafos).
  • Algoritmo de Prim (árbol de expansión mínima).
  • Heap Sort → O(n log n) con memoria extra O(1) (in-place).
  • Selección de k-ésimo elemento más grande/pequeño.
💡 Dato clave: Un heap se suele implementar en un arreglo (no con nodos enlazados) usando índices:
hijo_izq = 2*i + 1, hijo_der = 2*i + 2, padre = (i-1)//2.
TIPOS • MULTIRRAMA
💾 Árboles B (B-Tree y B+Tree)

Concepto

Árboles multirrama (cada nodo puede tener muchos hijos) diseñados específicamente para almacenamiento secundario (discos, SSDs).

Propiedad fundamental: Un nodo contiene múltiples claves ordenadas y múltiples punteros a hijos.
Ejemplo: Un nodo puede tener 100 claves y 101 hijos.

¿Por qué son tan eficientes en disco?

  • 🔹 Alto factor de ramificación → árbol muy ancho y bajo (altura pequeña).
  • 🔹 Menos niveles → menos accesos a disco (operación muy lenta).
  • 🔹 Los nodos tienen el tamaño de un bloque de disco (ej: 4KB), optimizando la lectura.

B-Tree vs B+Tree

B-Tree

Los datos (claves) están en todos los nodos (internos y hojas).

B+Tree (el más usado)

  • ✅ Los datos solo en las hojas.
  • ✅ Los nodos internos son solo índices.
  • ✅ Las hojas están enlazadas → recorridos por rango muy rápidos.
USADO EN MySQL (InnoDB), PostgreSQL, MongoDB, sistemas de archivos (NTFS, ext4).
TIPOS • ESPECIALES
🔤 Trie (Árbol de Prefijos)

Concepto

Un árbol cuyos nodos representan caracteres de una cadena. La raíz está vacía y cada camino desde la raíz forma un prefijo.

Propiedad clave: La búsqueda de una palabra de longitud L toma O(L).
¡Independiente de cuántas palabras haya en el árbol!

Intuición

En lugar de comparar la palabra completa con muchas otras (como en un hash o BST), el Trie la descompone en caracteres y sigue un camino único.

COMPLEJIDAD Inserción, búsqueda, eliminación: O(L) donde L = longitud de la palabra.
DESVENTAJA Puede consumir mucha memoria si hay muchos prefijos diferentes.

Aplicaciones Reales

  • 🔍 Autocompletado (Google, editores de código).
  • ✏️ Correctores ortográficos (sugerir palabras similares).
  • 🌐 Enrutamiento de IP (búsqueda de prefijo más largo).
  • 📂 Directorios y sistemas de archivos.
  • 🧬 Análisis de secuencias en bioinformática.
// Ejemplo: {casa, carro, caso, cama}
(raíz)
└── c
    └── a
        ├── s → a (casa)
        ├── s → o (caso)
        ├── r → r → o (carro)
        └── m → a (cama)
TIPOS • AVANZADOS
🧩 Otros Árboles Especiales

🌊 Splay Tree

Concepto: Árbol que se auto-reorganiza moviendo el nodo accedido a la raíz (splay).

  • 🔹 Propiedad: Los nodos más accedidos quedan cerca de la raíz.
  • 🔹 Uso: Cachés, compresores de datos.
  • 🔹 Amortizado: O(log n) por operación.

📐 Árbol Cartesiano

Concepto: Árbol binario donde se cumple Heap Property por una clave y BST Property por otra.

  • 🔹 Uso: Estructuras de datos geométricos, Treaps (mezcla de BST y Heap).
  • 🔹 Construcción en O(n).

📊 Árbol Fenwick (BIT)

Concepto: Árbol implícito en un arreglo que permite sumas de prefijos y actualizaciones eficientes.

  • 🔹 Operaciones: O(log n) para consulta y actualización.
  • 🔹 Uso: Estadísticas de frecuencias, inversiones, conteo de rangos.
💡 Nota: Cada uno de estos árboles resuelve un problema específico con restricciones particulares de tiempo y espacio.
ALGORITMOS
📝 Recorridos en Árboles Binarios

Los tres recorridos clásicos (DFS) con su propósito conceptual:

1. Pre-orden (Raíz → Izq → Der)

Propósito: Crear una copia exacta del árbol o generar un prefijo (serialización).

Ejemplo: 10, 5, 2, 7, 15, 12, 20

2. In-orden (Izq → Raíz → Der)

Propósito: Obtener los elementos en orden ascendente (solo en BST).

Ejemplo: 2, 5, 7, 10, 12, 15, 20 (ordenado)

3. Post-orden (Izq → Der → Raíz)

Propósito: Liberar memoria (borrar hijos antes que el padre) o evaluar expresiones.

Ejemplo: 2, 7, 5, 12, 20, 15, 10

class Nodo:
    def __init__(self, valor):
        self.valor = valor
        self.izq = None
        self.der = None

# Pre-orden
def preorden(nodo):
    if nodo:
        print(nodo.valor)
        preorden(nodo.izq)
        preorden(nodo.der)

# In-orden
def inorden(nodo):
    if nodo:
        inorden(nodo.izq)
        print(nodo.valor)
        inorden(nodo.der)

# Post-orden
def postorden(nodo):
    if nodo:
        postorden(nodo.izq)
        postorden(nodo.der)
        print(nodo.valor)

DECISIÓN DE INGENIERÍA
🎯 ¿Cuál Árbol Elijo?
La pregunta correcta no es "¿cuál es el mejor?", sino "¿cuál es el más adecuado para mi contexto?"

Guía Rápida

  • 🔹 Strings y prefijos → Trie
  • 🔹 Máximo/Mínimo rápido → Heap
  • 🔹 Datos en disco (GB/TB) → B-Tree / B+
  • 🔹 Muchas inserciones/eliminaciones → Rojo-Negro
  • 🔹 Muchas búsquedas, pocos cambios → AVL
  • 🔹 Acceso a elementos recientes → Splay
  • 🔹 Sumas de prefijos en arreglos → Fenwick

Filosofía del Experto

"Un árbol no es mejor que otro; es más adecuado para un contexto específico."

Recuerda: La elección correcta considera:

  • 📊 Volumen de datos (memoria vs disco).
  • ⚡ Frecuencia de operaciones (lectura vs escritura).
  • 🔍 Tipo de consulta (exacta, rango, prefijo).
  • 💾 Restricciones de memoria.
🌟 Mastery Tip: Conoce al menos 3 árboles en profundidad y podrás resolver el 90% de los problemas.
